# Техническое задание: шаблон интерактивного языкового курса на GitHub Pages

## Стек технологий

| Компонент | Технология |
|---|---|
| Разметка | HTML5 |
| Стили | CSS3 (Custom Properties, Grid, Flexbox) |
| Логика | Vanilla JavaScript (ES6+) |
| Озвучивание | Web Speech API (SpeechSynthesis) |
| Распознавание речи | Web Speech API (SpeechRecognition) |
| Хранение прогресса | localStorage |
| Хостинг | GitHub Pages |
| Версионирование | Git |
| Контент | JSON (самодостаточные файлы уроков) |
| Внешние зависимости | **Нет** (только встроенные API браузера) |

---

## 1. Назначение документа

Документ описывает архитектуру, формат данных и процесс создания интерактивного языкового курса, развёртываемого как статический сайт на GitHub Pages. Шаблон предназначен для воспроизведения курса любого языка с минимальной заменой контента.

**Базовая реализация:** курс «Испанский в диалогах» (espanol-en-dialogos, 55 диалогов, A2–C1).

### 1.1. Область применения

- Любой иностранный язык
- Уровни A1–C1
- Индивидуальное или групповое обучение
- Самостоятельное изучение или работа с преподавателем

### 1.2. Ключевые определения

| Термин | Описание |
|---|---|
| Диалог / Урок | Законченный учебный блок из 7 шагов |
| Часть | Тематический блок из нескольких диалогов |
| Шаг | Один этап урока (лексика, грамматика, диалог и т.д.) |
| Shadowing | Повторение фразы за диктором с записью/распознаванием |
| Roleplay | Ролевая игра: пользователь говорит за персонажа |

## 2. Системные требования

- Любой современный браузер (Chrome, Edge, Firefox, Safari) — для базовой функциональности
- **Chrome или Edge** — для распознавания речи (Shadowing, Roleplay)
- Включённый JavaScript
- Микрофон — для упражнений на говорение
- HTTPS (GitHub Pages предоставляет автоматически) или localhost — для SpeechRecognition

### 2.1. Ограничения

- Нет серверного кода
- Нет внешних API (кроме встроенных в браузер)
- Нет билд-шагов
- SpeechRecognition требует подключения к интернету
- TTS работает офлайн с системными голосами
- Базовая функциональность (чтение, тесты, прогресс) работает полностью офлайн

## 3. Архитектура приложения

### 3.1. Принцип работы

Две HTML-страницы:
- **index.html** — лендинг со списком частей и диалогов, прогресс-бар
- **lesson.html?id=N** — проигрыватель одного урока по его ID

Контент хранится в JSON-файлах в папке `data/`. Управление состоянием — через localStorage. Озвучивание — через Web Speech API.

### 3.2. Структура репозитория

```
{course-name}/
├── index.html              # Лендинг (список уроков)
├── lesson.html             # Проигрыватель урока
├── README.md               # Описание курса
├── favicon.ico             # Иконка вкладки
├── logo.png                # Логотип курса
├── css/
│   └── style.css           # Все стили (единый файл)
├── js/
│   ├── app.js              # Роутинг, инициализация
│   ├── lesson-player.js    # 7 шагов урока
│   ├── games.js            # Упражнения (6 типов)
│   ├── speech.js           # TTS + SpeechRecognition
│   └── progress.js         # localStorage
└── data/
    ├── dialogs.json        # Метаданные курса (все уроки)
    ├── characters.json     # Описание персонажей
    ├── dialog-01.json      # Урок 1
    ├── dialog-02.json      # Урок 2
    └── ...                 # Остальные уроки
```

### 3.3. Компоненты JS

| Файл | Назначение | Ключевые функции |
|---|---|---|
| `app.js` | Инициализация приложения, загрузка данных, роутинг | `loadDialog(id)` |
| `lesson-player.js` | Управление шагами урока | `LessonPlayer.init(dialog)`, `goToStep(n)`, 7 рендер-методов |
| `games.js` | Игровые механики упражнений | `memoryGame()`, `fillGap()`, `shadowingPlayer()`, `roleplayGame()`, `createDialogue()`, `quiz()` |
| `speech.js` | Синтез и распознавание речи | `Speech.say(text, lang, rate)`, `Speech.recognize(expected, callback)` |
| `progress.js` | Чтение/запись прогресса | `Progress.get(id)`, `Progress.set(id, data)`, `getStats()` |

### 3.4. Поток данных

```
index.html
  → fetch('data/dialogs.json')
  → рендер списка частей и диалогов
  → клик по диалогу → lesson.html?id=N

lesson.html
  → app.js: loadDialog(N)
    → fetch('data/dialog-NN.json')
    → LessonPlayer.init(dialog)
      → восстановление прогресса из localStorage
      → рендер текущего шага
      → навигация (Назад/Далее) → сохранение шага
      → завершение → отметка completed
```

## 4. Формат данных

### 4.1. dialogs.json — индекс курса

```json
[
  {
    "id": 1,
    "title": "Javier conoce a Ana",
    "titleRu": "Хавьер знакомится с Анной",
    "part": 1,
    "partTitle": "Базовый уровень (A2)",
    "level": "A2",
    "theme": "Presentaciones"
  }
]
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | number | Уникальный номер урока (1–N) |
| `title` | string | Название на целевом языке |
| `titleRu` | string | Название на языке инструкций |
| `part` | number | Номер части (1–6) |
| `partTitle` | string | Название части |
| `level` | string | Уровень сложности (A2, B1, C1 и т.д.) |
| `theme` | string | Тема урока |

### 4.2. dialog-NN.json — полный урок

```json
{
  "id": 1,
  "title": "Javier conoce a Ana",
  "titleRu": "Хавьер знакомится с Анной",
  "part": 1,
  "level": "A2",
  "theme": "Presentaciones (Знакомство)",
  "characters": ["javier", "ana"],
  "objectives": [
    "Научиться представлять себя"
  ],
  "grammar": {
    "ser": {
      "yo": { "es": "soy", "ru": "я есть" },
      "tu": { "es": "eres", "ru": "ты есть" }
    }
  },
  "vocabulary": {
    "greetings": [
      { "es": "Hola", "ru": "Привет" }
    ],
    "questions": [
      { "es": "¿Cómo te llamas?", "ru": "Как тебя зовут?" }
    ]
  },
  "dialogue": [
    { "speaker": "Javier", "es": "Hola, ¿qué tal?", "ru": "Привет, как дела?" }
  ],
  "exercises": [],
  "homework": [
    "5 мин: Повторить слова"
  ]
}
```

#### 4.2.1. Поля верхнего уровня

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | number | да | ID урока (совпадает с номером в `dialogs.json`) |
| `title` | string | да | Название на целевом языке |
| `titleRu` | string | да | Название на языке инструкций |
| `part` | number | да | Номер части |
| `level` | string | да | Уровень |
| `theme` | string | да | Тема |
| `characters` | string[] | нет | Массив ID персонажей (из `characters.json`) |
| `objectives` | string[] | да | Цели урока |
| `grammar` | object | да | Грамматические правила (см. ниже) |
| `vocabulary` | object | да | Группы лексики (см. ниже) |
| `dialogue` | object[] | да | Строки диалога |
| `exercises` | object[] | да | Массив упражнений (см. раздел 6) |
| `homework` | string[] | да | Домашнее задание |

#### 4.2.2. grammar

Ключи — названия правил (например: `ser`, `estar`, `presente`). Значения — объект с лицами:

```json
"ser": {
  "yo":       { "es": "soy",    "ru": "я есть" },
  "tu":       { "es": "eres",   "ru": "ты есть" },
  "el_ella":  { "es": "es",     "ru": "он/она есть" },
  "nosotros": { "es": "somos",  "ru": "мы есть" },
  "vosotros": { "es": "sois",   "ru": "вы есть" },
  "ellos_ellas": { "es": "son", "ru": "они есть" }
}
```

Допускается `{ "es": "string", "ru": "string" }` для каждой ячейки. Для простых правил — `"es": "string"`.

#### 4.2.3. vocabulary

Объект, где ключи — названия групп (например: `greetings`, `verbs`, `places`), значения — массивы пар:

```json
{
  "greetings": [
    { "es": "Hola", "ru": "Привет" }
  ]
}
```

#### 4.2.4. dialogue

Массив строк диалога:

```json
[
  { "speaker": "Javier", "es": "...", "ru": "..." }
]
```

| Поле | Тип | Описание |
|---|---|---|
| `speaker` | string | Имя персонажа (совпадает с `name` в коде отображения) |
| `es` | string | Текст на целевом языке |
| `ru` | string | Перевод на язык инструкций |

### 4.3. characters.json

```json
[
  { "id": "javier", "es": "Javier – estudiante de Italia", "ru": "Хавьер – студент из Италии" }
]
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный ID для ссылок из `dialog-NN.json` |
| `es` | string | Описание на целевом языке |
| `ru` | string | Описание на языке инструкций |

## 5. 7-шаговая структура урока

Каждый урок состоит из 7 обязательных шагов, проходимых последовательно:

### Шаг 1. Подготовка
- Приветствие, постановка контекста темы
- Цели урока
- Информация о диалоге (уровень, тема)
- Список персонажей с кратким описанием
- Кнопка «Начать урок →»

**Механика:** статический рендер из JSON-полей `objectives`, `theme`, `level`, `characters` + захардкоженные интро по темам.

### Шаг 2. Лексика (Словарный запас)
- Карточки слов: испанское слово + перевод + кнопка 🔊
- Игра «Соедини пары» (Memory Match): две колонки, клик — выбор пары

**Механика:** все элементы `vocabulary` объединяются в один массив, рендерятся карточками. Игра — из первых 8 пар (или всех).

### Шаг 3. Грамматика
- Таблицы спряжения глаголов/правил с построчным переводом
- Кнопка «Прослушать формы»
- Упражнение «Вставь правильную форму» (Fill-Gap)

**Механика:** динамическая генерация HTML-таблиц из `grammar`. Fill-Gap берёт `exercises[type=fill-gap]`.

### Шаг 4. Диалог
- Построчный текст диалога: персонаж | оригинал | перевод
- Кнопка «Прослушать весь диалог»
- Shadowing-плеер: TTS → кнопка «Повторить» → SpeechRecognition → оценка

**Механика:** `dialogue` рендерится сеткой из 3 колонок. Shadowing — пофразово с навигацией.

### Шаг 5. Говорение (Практика говорения)
- «Создай свой диалог»: заполнение пропусков → сохранение в localStorage → TTS
- Roleplay: роль пользователя → контекст оппонента → распознавание реплик

**Механика:** Create Dialogue берёт `exercises[type=create-dialogue].template`. Roleplay фильтрует `dialogue` по `speaker === role`.

### Шаг 6. Тестирование
- Вопросы с вариантами ответа (Quiz)
- Подсветка правильного/неправильного ответа
- Счёт очков

**Механика:** `exercises[type=quiz].questions`. Кнопки-варианты → проверка по `answer` (индекс).

### Шаг 7. Рефлексия
- Поздравление с завершением урока
- Чек-лист пройденных этапов
- Домашнее задание (из `homework`)
- Кнопки: «Пройти заново», «На главную»

### Навигация

- **Индикатор шагов** — 7 точек-кружков над контентом: серая (не начат), зелёная (пройден), красная (текущий)
- **Кнопки** — «← Назад» и «Далее →» под контентом
- **Сохранение** — при каждом шаге `lastStep` пишется в localStorage
- **Возврат** — при открытии урока восстанавливается последний шаг

## 6. Типы упражнений

### 6.1. Memory Match (Соедини пары)
- **Тип:** `memory`
- **Поле в JSON:** `exercises[type=memory].pairs` — массив `{ es, ru }`
- **Механика:** две колонки со случайным порядком. Клик по слову в левой, затем по переводу в правой. Совпадение по индексу. Зелёная подсветка при успехе, красная при ошибке.
- **Завершение:** все пары найдены → сообщение об успехе.

### 6.2. Fill-Gap (Вставь пропуск)
- **Тип:** `fill-gap`
- **Поле в JSON:** `exercises[type=fill-gap].sentences` — массив `{ text, answer, options[] }`
- **Механика:** текст с `___` заменяется на `<select>` с вариантами. Кнопка «Проверить» подсвечивает правильные (зелёный) и неправильные (красный + верный ответ) выборы.

### 6.3. Shadowing (Повторение)
- **Тип:** `shadowing`
- **Поле:** не требует дополнительных данных (использует `dialogue`)
- **Механика:** показывается одна фраза → кнопка «🔊 Слушать» (TTS) → кнопка «🎤 Повторить» → SpeechRecognition → процент совпадения слов. Если <80%, предлагает повторить.

### 6.4. Roleplay (Ролевая игра)
- **Тип:** `roleplay`
- **Поле в JSON:** `exercises[type=roleplay]` — массив с `{ title, role, description }`
- **Механика:** отфильтрованные реплики персонажа. Показывается контекст (реплика собеседника) + ожидаемая реплика пользователя. Кнопка «🎤 Скажи это» → SpeechRecognition → оценка.

### 6.5. Create Dialogue (Создай диалог)
- **Тип:** `create-dialogue`
- **Поле в JSON:** `exercises[type=create-dialogue].template` — массив `{ label, es }` (es содержит `___`)
- **Механика:** поля ввода с подсказками. Заполнение → сохранение в `localStorage` (ключ `{course}_my_dialogues`) → TTS готового диалога.

### 6.6. Quiz (Тест)
- **Тип:** `quiz`
- **Поле в JSON:** `exercises[type=quiz].questions` — массив `{ question, ru, options[], answer }` (answer — индекс правильного варианта)
- **Механика:** вопросы по одному. Выбор варианта → блокировка → подсветка правильного/неправильного. Счёт сохраняется в localStorage.

## 7. Озвучивание и распознавание речи

### 7.1. SpeechSynthesis (TTS)

```js
Speech.say(text, lang = 'es-ES', rate = 0.72)
```

- `text` — предварительно очищается от служебных символов через `cleanForTTS()`
- `lang` — код языка (например, `es-ES`, `fr-FR`, `de-DE`, `ru-RU`)
- `rate` — скорость (0.72 для медленного учебного темпа)

### 7.2. SpeechRecognition

```js
Speech.recognize(expectedText, callback(percent, transcript))
```

- `lang` — язык распознавания (должен совпадать с языком курса)
- `percent` — процент совпадения слов (0–100) или `null` при ошибке
- Алгоритм сравнения: пересечение множеств слов ожидаемого и распознанного текста
- Поддерживается только в Chrome/Edge

### 7.3. Очистка текста для TTS

Удаляются:
- Суффиксы родовых окончаний через `/` (например, `/o`, `/a`, `/os`, `/as`)
- Лишние пробелы

## 8. Прогресс и хранение данных

### 8.1. Прогресс уроков

**Ключ localStorage:** `{prefix}_progress` (например, `espanol_progress`)

```json
{
  "dialog-1": { "completed": true, "score": 85, "lastStep": 7 },
  "dialog-2": { "completed": false, "score": null, "lastStep": 3 }
}
```

### 8.2. Сохранённые диалоги пользователя

**Ключ localStorage:** `{prefix}_my_dialogues`

```json
[
  { "date": "2026-06-27T...", "text": "Me llamo...", "values": ["..."] }
]
```

### 8.3. API модуля Progress

| Метод | Описание |
|---|---|
| `Progress.get(dialogId)` | Получить прогресс по ID (по умолчанию `{ completed: false, score: null, lastStep: 0 }`) |
| `Progress.set(dialogId, updates)` | Обновить поля прогресса |
| `Progress.getStats()` | Получить `{ total, completed }` для отображения на главной |

## 9. Адаптация курса под новый язык

Для создания нового курса на основе шаблона нужно заменить:

### 9.1. Контент
1. Все файлы в `data/` — новые диалоги, лексика, грамматика
2. `characters.json` — персонажи нового курса
3. `favicon.ico` и `logo.png` — брендинг

### 9.2. Код (минимальные изменения)
- В `speech.js` — заменить умолчания `es-ES` на код нужного языка
- В `lesson-player.js`:
  - Обновить `themeIntro` — контекстные вступления для тем нового курса
  - Обновить захардкоженные подсказки в грамматике (секция `grammar` рендера, строки с `d.id === 1 ? ...`)
- В `css/style.css` — обновить CSS-переменные под бренд:
  ```css
  :root {
    --primary: #E63946;      /* Основной цвет курса */
    --bg: #FFF8F0;           /* Фон */
    --accent: #F4A261;       /* Акцент */
    --text: #264653;         /* Цвет текста */
    --success: #2A9D8F;      /* Успех */
    --error: #e76f51;        /* Ошибка */
  }
  ```
- В `index.html` — заголовок, описание, ссылки, футер
- В `lesson.html` — заголовок, обратная ссылка

### 9.3. Настройки (без изменений кода)
- `progress.js:STORAGE_KEY` — изменить префикс для изоляции данных нового курса
- GitHub Pages настраивается в Settings репозитория

## 10. Деплой на GitHub Pages

### 10.1. Требования к репозиторию
- Публичный репозиторий на GitHub
- В корне репозитория — `index.html` (точка входа)
- Никаких билд-скриптов, никаких зависимостей

### 10.2. Настройка GitHub Pages
1. Репозиторий → Settings → Pages
2. Source: `Deploy from a branch`
3. Branch: `main` (или `gh-pages`), folder: `/ (root)`
4. URL: `https://{username}.github.io/{repo-name}/`

### 10.3. Кастомный домен (опционально)
1. Добавить CNAME-запись у регистратора
2. Создать файл `CNAME` в корне репозитория с содержимым `yourdomain.com`
3. В Settings → Pages → Custom domain: `yourdomain.com`
4. Включить Enforce HTTPS

### 10.4. CI/CD (опционально)
GitHub Actions не требуется для базового деплоя — push в main автоматически триггерит Pages. При желании можно добавить линтер для JSON:

```yaml
# .github/workflows/validate.yml
name: Validate
on: [push]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Validate JSON
        run: |
          for f in data/*.json; do
            jq empty "$f" || exit 1
          done
```

## 11. Ограничения и рекомендации

### 11.1. Браузерные ограничения
- **SpeechRecognition** — только Chrome/Edge через HTTPS или localhost
- **SpeechSynthesis** — доступен во всех браузерах, голоса зависят от ОС
- В Firefox и Safari упражнения на говорение доступны в режиме самопроверки (без распознавания)

### 11.2. Производительность

- Карточки, кнопки и инпуты адаптивного размера
